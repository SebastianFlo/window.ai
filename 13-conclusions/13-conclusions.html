Conclusions

Pros

  • Powered by Google
  • Free (for now at least)
  • Model is in the browser, shared by multiple websites
  • Uniform JS API
  • Simplified language model (The complicated stuff is abstracted in the browser)
  • Privacy: Local processing of sensitive data - data should never leave the device (in theory)
  • Speed: Local device should be faster in case of slow network
  • Offline usage (remote areas, festivals etc...)
  • Powered by device energy: More sustainable (no coal powered AI needed)

Cons

  • Powered by Google
  • Very limited
  • No up to date information (last model from september 2024)
  • Only in chrome for now. Will other browsers use this or their own, with their own version (canAIuse .com ? :D)
  • Experimental, so api changes very often

Predictions

  • These local models will only get better. Large models might all ship browser versions
  • Could become user defined (based on browser, or what the model the user decides to load in their browser)
  • Endless applications, like summarising data, extracting payloads from text, validation
  • AI-enhanced content consumption: Including summarization, translation, categorization, characterization, and as a knowledge provider
  • AI-supported content creation: Such as writing assistance, proofreading, grammar correction, and rephrasing
